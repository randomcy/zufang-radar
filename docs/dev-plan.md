## 兜底原则

如果某个 phase 严重超时，按优先级砍：
1. **最先砍**：功能二的高德地图复杂交互 → 退化为静态地图截图 + 站点列表
2. **再砍**：场景 B 的 Bradley-Terry → 退化为简单胜场计数
3. **绝不砍**：功能三的小区体检报告（这是 demo 灵魂）
4. **绝不砍**：场景 A 的偏好画像（这是算法亮点）
